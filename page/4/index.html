排列

从 n 各不同元素中，任取 m (m <= n) 个元素，按照一定的顺序排列，叫做从 n 个不同元素中 取出 m 个元素的一个排列

\[{\color{Blue} A_{n}^{m} = \frac{n!}{(n-m)!}}\]

组合

从 n 个不同元素中，任取 m (m <= n) 个元素组成一组(不考虑顺序)，叫做从 n 个不同元素中 取出 m 个不同元素的一个组合

\[{\color{Blue} C_{n}^{m} = \frac{n!}{(n-m)!*m!}}\]

利用分步计数原理有：

  1. 先给①涂色有 \({\color{Blue} C_{5}^{1} = 5}\) 种方法
  2. 再给②涂色有 \({\color{Blue} C_{4}^{1} = 4}\) 种方法
  3. ③有 \({\color{Blue} C_{3}^{1} = 3}\) 种
  4. ④有 \({\color{Blue} C_{2}^{1} = 2}\) 种

故总共有 5 x 4 x 3 x 2 = 240 种方法

2. 扇形上色问题

图2
  1. 当A、C、E着同一种颜色时，有 \({\color{Blue} C_{4}^{1} = 4}\) 种着色方案 ，此时，B、D、F各有 \({\color{Blue} C_{3}^{1} = 3}\) 种着色方案。故此时有 4x3x3x3=108 种方法
  2. 当A、C、E着两种不同颜色时，有 \({\color{Blue} C_{4}^{2} A_{3}^{3} = 36}\) 种方法，此时， B、D、F有 3x2x2=12 方法。故此时有 36x12=432 种方法
  3. 当A、C、E着三种不同颜色时，有 \({\color{Blue} A_{4}^{3} = 24}\) 种方法，此时， B、D、F各有2种方法。故此时有 24x2x2x2=192 种方法

故此时总共有 108 + 432 + 192 = 732 种方法

Kruskal 概述

Kruskal算法，是用来求加权连通图的最小生成树的算法。

它的基本思想是：按照权值从小到大的顺序选择n-1条边，并保证这n-1条边不构成回路（环）。

具体做法是：首先构造一个只含n个顶点的森林， 然后依权值从小到大从连通网中选择边加入到森林中， 并使森林中不产生回路，直至森林变成一棵树为止。

更多关于kruskal的知识欢迎查看维基百科-Kruskal's algorithm ，本文主要是记录并查集这个数据结构怎么用在Kruskal求解最小生成树的问题上。

并查集（union-find set）概述

union-find set 翻译成中文就是并查集, 并查集是一种树型的数据结构，用于处理一些不相交集合（disjoint sets）的合并及查询问题。

并查集的主要操作：

1. 初始化：把每个点所在集合初始化为其自身。

2. 查找：查找元素所在的集合，即根节点。

3. 合并：将两个元素所在的集合合并为一个集合。

并查集判断无向图是否有环

如上图, 如何判断该图是否有环呢？首先你可能会想到用递归的方式去做所以你用了DFS， 没错DFS确实可以做到，但是复杂度过高，那么能否用并查集来判断无向图是否有环呢？ 答案是肯定的。

首先规定如果两个点之间有边相连，如果这两个点不属于同一个集合，那就将他们合并成一个集合

  1. 我们把每个点看成独立的集合{0} ，{1}， {2}
  2. 对于边0-1，直接将这两个点代表的集合合并{0， 1}, 让1来当父节点
  3. 对于边1-2， 它们分别属于不同的集合，合并集合之后是{1，2}，让2来当父节点， 这时候有0->1->2 即 0的祖先节点是2
  4. 对于边0-2，他们属于一个集合，因为他们有着共同的祖先2， 这就说明0-2之间在没有0-2这条边之前已经连通了， 如果加上这条边的话那从0到2就有两条路径可达，就说明存在环了

xmnlp — A Lightweight Chinese NLP Toolkit.

发表于 2018-02-16 更新于 2021-03-07
本文字数： 1.5k 阅读时长 ≈ 1 分钟

python下NLP工具有很多 jieba, nltk, ltp 等， 虽然他们很强大，但是提供的功能比较分散，而且通常模型比较大。为了方便平时的处理工作，我尝试找了一些集成工具包，发现snownlp还可以，它的分词是基于TnT的，总得来说分词效果逊色于基于词典的分词（比如jieba）。所以决定自己写一个包xmnlp，主打轻量快捷。

功能

  1. 中文分词 & 词性标注： 基于词典构建DAG图，然后采用动态规划的思想求得最大概率路径（jieba分词采用了反向输出，我采用了正向加权反向输出的方式，使得正反向共同影响分词效果），对于未登录词采用HMM+Viterbi处理

  2. 文本纠错：采用了bi-gram + levenshtein实现

  3. 文本摘要 & 关键词提取：textrank

  4. 情感分析：naive bayes

  5. 文本转拼音：Trie 树检索
